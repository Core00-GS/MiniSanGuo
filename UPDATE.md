《迷你三国》版本履历（从 0.0.0.0 开始）
0.0.0.0｜原始原型版

这就是你最早贴出来的那份代码，核心特点是“能跑、能点、能打”。

已具备：

Tkinter 主窗口
我方兵力 / 敌方兵力
日期系统
体力系统
资源系统
内政等级
住宅恢复体力
征兵
内政升级：
商业
农业
治安
城墙修复
战斗地图
我方军营、敌方军营、随机村庄
点击城市后执行攻击
占领村庄 / 敌方军营
简单的城市重叠检测

这一版的定位是：
“单机策略游戏的最初可玩雏形”

0.0.0.1｜界面刷新与状态同步加强

这一阶段的重点是让数据变化后，界面能跟上。

新增重点：

日期显示刷新
体力显示刷新
资源文字刷新
内政数值刷新
我方兵力显示刷新
城市占领后数值同步

和 0.0.0.0 相比，这一步不是加玩法，而是让游戏“看起来更完整”。

0.0.0.2｜战场表现增强版

这一阶段开始从“只有几个圆圈”往“地图玩法”靠近。

新增重点：

战场地图更明确
据点颜色区分更清楚
我方 / 敌方 / 中立的视觉差异
占领后显示“我方占领”
战斗后重新绘制地图

这一阶段主要解决的是：
“打完以后，画面要能看出来变化”

0.0.0.3｜内政循环稳定版

这一阶段重点是资源和行动节奏。

新增重点：

征兵有体力成本
提升商业消耗金钱
提升农业消耗木材
提升治安消耗粮食
修复城墙消耗金钱
建设住宅可回满体力并推进日期

和最初版本相比，这一版更像一个完整的“日常循环”：
内政 → 积累资源 → 恢复体力 → 继续作战

0.0.0.4｜多场景界面版

这一阶段开始把界面从“一个页面塞全部内容”变成“分场景”。

新增重点：

主界面
战场界面
不同界面之间切换
每个界面有自己的信息区、按钮区、地图区

这一阶段的意义很大，因为它让项目从“教学型 demo”升级为：
“真正能继续扩展的游戏框架”

0.0.0.5｜武将数据外置版

这一阶段开始引入 generals.json。

新增重点：

武将数据不再写死在程序逻辑里
使用 JSON 保存武将资料
支持加载 / 保存武将数据
武将拥有固定属性：
武力
智力
政治
统率
忠诚
兵力
存活状态

这一阶段的核心变化是：
从“只有兵力数值”变成“有武将角色的数据系统”

0.0.0.6｜武将选择与多将出征版

这一阶段是玩法变化最大的地方之一。

新增重点：

战前可选择出战武将
支持多名武将同时出征
右侧武将列表
武将详情面板
出战确认机制
战斗时根据武将属性计算攻击力

攻击力不再只是单纯看兵力，而是开始参考：

兵力
武力
统率
出战人数

这一步以后，游戏不再是“军营对军营”的简单碰撞，而是：
“谁带谁上阵” 的策略系统

0.0.0.7｜技能系统版

这一阶段把武将个性加进来了。

新增重点：

刘备：仁德
关羽：武圣
张飞：猛将
诸葛亮：神机
其他武将：无技能

技能效果开始影响战斗：

武圣：增强攻击
猛将：增强攻击，但损耗更高
仁德：降低损耗
神机：减少损失，偏向保全

这一阶段的意义是：
武将不只是数值，开始有战术差异

0.0.0.8｜俘虏与招降版

这一阶段开始有“战后处理”。

新增重点：

战胜后可俘虏敌将
俘虏后可选择招降
招降成功：转为我方，忠诚度重置
招降失败：处斩

这让战斗结果不再只是“输赢”，而是：
“战后还能改变势力结构”

0.0.0.9｜战报与状态记录版

这一阶段加强了反馈体验。

0.0.1.1｜战场地图“冻结渲染版”（重要架构优化）

这一版本开始解决你提到的核心问题：

❗“出战后地图刷新导致占领村庄消失”

🔧 核心改动：
战场地图 不再重建 cities 数据
create_battle_scenario() 改为：
只在“进入战场时生成一次”
后续战斗 只更新状态，不重置地图
show_battle_screen() 不再调用“重新生成地图数据”
🧠 本质变化：

以前：

每次刷新 = 重建世界 ❌

现在：

世界固定 + 状态变化 ✔
